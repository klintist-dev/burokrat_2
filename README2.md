Раздел 3. Создание бота
Шаг 1. Подготовка рабочего пространства

Прежде чем писать код, нужно подготовить рабочее пространство. В Python для этого принято использовать виртуальное окружение — это как отдельный «чемодан» со всеми инструментами для проекта, чтобы они не мешали другим проектам.

Я работаю на MacOS, в качестве среды разработки использую PyCharm. Если у вас Windows или другая IDE — не беда, команды будут почти такими же, за исключением пары нюансов, которые я буду отмечать.

    Создаем папку проекта и переходим в нее (В PyCharm это можно сделать через интерфейс, но я предпочитаю терминал — так понятнее, что происходит "под капотом".):

    mkdir burokrat_bot
    cd burokrat_bot

    Создаем виртуальное окружение:

    python3 -m venv .venv

    # Активируем его (на Mac/Linux)
    source .venv/bin/activate

    # Для Windows команда была бы другой: venv\Scripts\activate

    Что здесь происходит:

    mkdir burokrat_bot — создаем папку (make directory)

    cd burokrat_bot — заходим в нее (change directory)

    python3 -m venv .venv — создаем виртуальное окружение с именем .venv

    source .venv/bin/activate — активируем его. После активации в терминале появится (venv) в начале строки — значит, всё хорошо.

    После активации все библиотеки будут устанавливаться только в этот проект, а не глобально в систему. Это хорошая практика.

    Важно для Mac: если у тебя несколько версий Python, убедись, что python3 ведет на нужную. Проверить можно командой which python3.

    В PyCharm: нужно будет указать это виртуальное окружение как интерпретатор проекта (File → Settings → Project → Python Interpreter → Add Local).

Создаем файл с секретами .env

Важно! Этот файл нельзя публиковать в Git. Создаем его прямо в корне папки проекта:

🔍 Что такое BOT_DEBUG и зачем он нужен

BOT_DEBUG — это флаг режима отладки. В переводе с английского "debug" означает "отладка" — процесс поиска и исправления ошибок в программе.
Как это работает:

debug: bool = False  # По умолчанию режим отладки ВЫКЛЮЧЕН

    False = выключено (режим для обычных пользователей)

    True = включено (режим для разработчика)

Зачем это нужно в коде

Когда ты разрабатываешь бота, часто нужно видеть больше информации, чем видят обычные пользователи. Например:

Когда debug = False (для пользователей):

    Бот просто работает

    В консоли минимум информации

    Пользователь видит только ответы

Когда debug = True (для разработчика):

    В консоль выводятся подробные логи всех действий

    Показываются ошибки с полным стеком вызовов

    Можно видеть, какие запросы отправляются и какие ответы приходят

Шаг 4. Создаем файл для чтения настроек

Создаем файл config_reader.py для чтения настроек. Этот файл отвечает за чтение секретов из .env.

Почему нельзя просто написать токен в коде? Потому что тогда любой, кто увидит код, сможет украсть твоего бота.

Создаем папку bot и внутри нее файл config_reader.py:

mkdir bot
touch bot/config_reader.py

Теперь открываем bot/config_reader.py и пишем:

# ⬇️ Pydantic — библиотека для работы с данными. SecretStr — тип "секретная строка"
from pydantic import SecretStr

# ⬇️ BaseSettings — читает настройки из .env автоматически
from pydantic_settings import BaseSettings

# ⬇️ lru_cache — запоминает результат функции, чтобы не читать .env каждый раз
from functools import lru_cache

class BotConfig(BaseSettings):
    """Класс с настройками бота"""
    
    token: SecretStr                # SecretStr - при печати будет показывать '****'
    admin_id: int                   # Твой Telegram ID. Нужен для статистики
    bot_name: str = "БюрократЪ 2.0" # Имя бота. 
    debug: bool = False             # Режим отладки. True - будет печатать больше информации

    class Config:
        env_file = ".env"            # Где лежат секреты
        env_prefix = "BOT_"          # Все переменные начинаются с BOT_
        env_file_encoding = "utf-8"  # Кодировка файла (чтобы русские буквы читались)

# Декоратор. Запоминает результат функции после первого вызова
@lru_cache
    """
    Возвращает конфиг (объект с настройками).
    lru_cache гарантирует, что функция выполнится только 1 раз,
    а потом будет возвращать сохранённый результат.
    Это экономит время и ресурсы.
    """
def get_config() -> BotConfig:
    """Возвращает конфиг (кэшируется)"""
    return BotConfig()

Что здесь происходит простыми словами:

class BotConfig(BaseSettings)
	

мы создаём "коробку" с настройками. В этой коробке будут лежать: токен, admin_id и т.д.

token: SecretStr
	

говорим: "в коробке есть отделение для токена, и оно секретное"

env_prefix = "BOT_"
	

говорим: "когда будешь читать .env, ищи переменные, которые начинаются с BOT_"

@lru_cache
	

"запомни результат, чтобы не читать файл каждый раз"

🔐 Почему токены нельзя хранить в коде?

Представь, что ты написал токен прямо в коде и выложил на GitHub. Любой может: найти твоего бота, узнать токен, угнать бота (менять команды, отправлять спам). Поэтому мы храним токены в .env, а сам .env — в .gitignore (чтобы не попадал в git.
Шаг 5. Создаем главный файл для запуска бота

В папке bot создаем файл main.py:

touch bot/__main__.py

Теперь напишем код, который создает бота и запускает его:

# ⬇️ Подключаем библиотеку для асинхронного кода
import asyncio

# ⬇️ Подключаем библиотеку для логирования (записи событий).
import logging

# ⬇️ Из библиотеки aiogram импортируем два основных класса: Bot и Dispatcher
from aiogram import Bot, Dispatcher

# ⬇️ Импортируем настройки по умолчанию для бота
from aiogram.client.default import DefaultBotProperties

# ⬇️ Импортируем перечисление режимов форматирования текста.
from aiogram.enums import ParseMode

# ⬇️ Импортирует нашу собственную функцию get_config из файла config_reader.py.
from bot.config_reader import get_config

# Настраиваем логирование
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

# Объявляет главную функцию программы. Слово async говорит, что функция асинхронная.
async def main():
    """Главная функция запуска бота"""
    
    # Получаем конфиг с токенами
    config = get_config()
    
    # Создаем объект бота
    bot = Bot(
        token=config.token.get_secret_value(),
        default=DefaultBotProperties(parse_mode=ParseMode.HTML)
    )
    
    # Создаем диспетчер
    dp = Dispatcher()
    
    # Запускаем бота
    try:
        print("🚀 БюрократЪ 2.0 запущен!")
        await dp.start_polling(bot)
    finally:
        await bot.session.close()

if __name__ == "__main__":
    asyncio.run(main())

Что здесь происходит:

Строка
	

Что делает

import asyncio
	

Сокращение от Asynchronous I/O — асинхронный ввод-вывод. Библиотека позволяет программе делать несколько дел одновременно. Например, пока бот ждет ответ от сайта, он уже может отвечать другому пользователю. Без этого бот зависал бы на каждом запросе.

import logging
	

Это как бортовой самописец, записывает всё, что происходит с программой: когда запустилась, когда упала, какие были ошибки. Потом эти записи можно читать и понимать, что пошло не так.

from aiogram import Bot, Dispatcher
	

Основные классы для бота:

- Bot — это сам бот. Как пульт управления

- Dispatcher — это диспетчер. Как секретарь, который распределяет входящие сообщения по нужным обработчикам

from aiogram.client.default import DefaultBotProperties
	

Это как заводские настройки для бота — базовые параметры, с которыми он будет работать.

from aiogram.enums import ParseMode
	

Определяет, как бот будет понимать форматирование текста. Например, HTML — значит в тексте можно использовать теги типа <b>жирный</b>.

from bot.config_reader import get_config
	

Подключаем функцию, которая прочитает токен из секретного файла .env. Это как достать ключ из кармана.

logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s -          %(name)s - %(levelname)s -     %(message)s'
 )
	

Говорит программе: "Записывай все события уровня INFO и выше, и делай это в таком формате: время - имя программы - уровень важности - само сообщение". Фактически шаблон для записи событий.

async def main():
	

Это точка входа в программу. Как дверь в дом — всё начинается отсюда. async означает, что функция может "засыпать" и "просыпаться", ожидая ответов от других сервисов.

config = get_config()
	

Простыми словами: Достаем из кармана ключи и настройки, которые там лежали.

bot = Bot(

token=config.token.get_secret_value(),

default=DefaultBotProperties(parse_mode=ParseMode.HTML)

 )
	

Собираем самого бота и даем ему:

- токен — его удостоверение личности (достаем из настроек); Важно: get_secret_value() нужно, потому что token у нас типа SecretStr — он прячет настоящее значение при печати, но здесь мы его "распаковываем" для использования.

- режим форматирования HTML — чтобы он понимал красивое оформление текста

dp = Dispatcher()
	

Простыми словами: Нанимаем секретаря, который будет смотреть на входящие сообщения и решать: "Это команда /start — отправить к обработчику start, это вопрос — отправить к AI".

try:
     print("🚀 БюрократЪ 2.0        запущен!")
 

    await dp.start_polling(bot)
 finally:
     await bot.session.close()
	

try: Начинает блок обработки ошибок. Говорим: "Попробуем сделать что-то, и если случится ошибка — мы к этому готовы".

print: Выводим в консоль приветственное сообщение и просто радуемся, что бот заработал.

start_polling(bot)Запускает бесконечный цикл получения обновлений от Telegram. Бот начинает слушать Telegram. Он постоянно спрашивает: "Есть новые сообщения для меня?" — и когда они появляются, передает их диспетчеру. await означает, что программа может подождать, пока придет ответ, и заняться пока другими делами.

finally: Блок кода, который выполнится в любом случае — была ошибка или нет. Простыми словами: Что бы ни случилось (даже если бот упал), мы должны сделать вот это.

bot.session.close()Закрывает сессию бота, освобождая соединения. Простыми словами: Закрываем все открытые двери, выключаем свет и уходим. Чтобы не оставлять за собой мусор.

if name == "__main__":
	

Проверяем, запущен ли этот файл напрямую или импортирован как модуль. Простыми словами: Берет нашу асинхронную функцию и запускает ее в специальной среде, где она может спокойно ждать ответы и при этом не блокировать всю программу.
